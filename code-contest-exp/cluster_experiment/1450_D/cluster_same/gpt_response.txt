```python
import os
import sys

def generate_test_case(num_cases, max_n, filename):
    with open(filename, 'w') as f:
        f.write(f"{num_cases}\n")
        
        for _ in range(num_cases):
            # Large value of n to stress test the algorithm
            n = max_n
            f.write(f"{n}\n")
            
            # Generate an array [1, 2, 3, ..., n]
            arr = " ".join(str(i) for i in range(1, n + 1))
            f.write(arr + "\n")

def main():
    if len(sys.argv) != 2:
        print("Usage: python generate_tests.py <output_directory>")
        sys.exit(1)

    output_directory = sys.argv[1]

    if not os.path.exists(output_directory):
        os.makedirs(output_directory)

    # Adjust the variables below as needed
    num_test_files = 5
    num_cases_per_file = 10
    max_n = 300000

    for i in range(1, num_test_files + 1):
        filename = os.path.join(output_directory, f"test_{i:02d}.in")
        generate_test_case(num_cases_per_file, max_n, filename)

if __name__ == "__main__":
    main()
```

### Explanation:
- This script generates `num_test_files` test files, each containing `num_cases_per_file` test cases.
- Each test case consists of an array of length `max_n` with elements ranging from `1` to `n` in ascending order.
- This setup ensures that it puts significant stress on the algorithm due to the large input size, maximizing the chances of hitting a time limit exceeded (TLE) error if the algorithm isn't optimized.

### Usage:
1. Save the script to a file, e.g., `generate_tests.py`.
2. Make the script executable:
   ```sh
   chmod +x generate_tests.py
   ```
3. Run the script with an argument specifying the output directory for the test files:
   ```sh
   ./generate_tests.py /path/to/output_directory
   ```
   
This will create test files named `test_01.in`, `test_02.in`, ..., in the specified directory, each containing the generated test cases.
