Creating a Python script to generate test cases for the provided problem statement, especially to test the time limit for the given solution codes, is an intensive task. The test cases should aim to exhaust the solution codes and fit the specified input format.

Here's a Python script that generates test cases to stress test the solutions. The script accepts a directory argument and writes all the test cases into files like `test_01.in`, `test_02.in`, etc.

### Python Script to Generate Test Cases

```python
import os
import sys
import random

def generate_test_cases(num_cases, max_len, out_dir):
    random.seed(42)  # Fixed seed for reproducibility

    if not os.path.exists(out_dir):
        os.makedirs(out_dir)

    for i in range(1, num_cases + 1):
        filename = os.path.join(out_dir, f"test_{i:02d}.in")
        with open(filename, 'w') as f:
            # Number of test cases
            f.write("1\n")
            # Length of the string (n)
            n = max_len
            f.write(f"{n}\n")
            # Generate a random string of length n containing 'D' and 'K'
            s = "".join(random.choice("DK") for _ in range(n))
            f.write(f"{s}\n")

def main():
    if len(sys.argv) != 2:
        print("Usage: python generate_test_cases.py <output_directory>")
        return
    
    out_dir = sys.argv[1]

    num_cases = 100  # Number of test cases to generate
    max_len = 500000  # Maximum length of the string in each test case

    generate_test_cases(num_cases, max_len, out_dir)
    print(f"Generated test cases in directory: {out_dir}")

if __name__ == "__main__":
    main()
```

### Steps to Use the Script

1. **Save the Script**: Save the provided script into a file named `generate_test_cases.py`.

2. **Run the Script**: Execute the script from the command line and provide the output directory where you want the test cases to be saved.
   ```
   python generate_test_cases.py <output_directory>
   ```

### Example Usage
```sh
python generate_test_cases.py ./test_cases
```

This script will generate 100 test cases in the `./test_cases` directory, each file named `test_01.in`, `test_02.in`, ..., `test_100.in`. Each file will contain a single test case with the maximum string length (`500,000` characters) filled with random 'D' and 'K' characters, which should be sufficient to push the limits and check for potential time limit exceeded scenarios in the provided solution codes.
