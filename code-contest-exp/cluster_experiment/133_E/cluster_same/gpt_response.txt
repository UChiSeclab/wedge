```python
import os
import random

def generate_test_cases(directory, num_cases=10, max_length=100):
    if not os.path.exists(directory):
        os.makedirs(directory)

    for i in range(1, num_cases + 1):
        filename = os.path.join(directory, f'test_{i:02d}.in')
        with open(filename, 'w') as file:
            length = max_length
            n = min(50, length)

            # Generate a random sequence of 'F' and 'T'
            commands = ''.join(random.choices(['F', 'T'], k=length))
            
            file.write(f"{commands}\n")
            file.write(f"{n}\n")

            print(f"Generated {filename}")

if __name__ == "__main__":
    import argparse

    parser = argparse.ArgumentParser(description='Generate test cases for the turtle path problem.')
    parser.add_argument('directory', type=str, help='The directory to save the test cases')
    parser.add_argument('--cases', type=int, default=10, help='The number of test cases to generate')
    parser.add_argument('--max_length', type=int, default=100, help='The maximum length of commands')
    
    args = parser.parse_args()
    
    generate_test_cases(args.directory, args.cases, args.max_length)
```

### Explanation:
1. **Imports**: It imports necessary modules (`os` for filesystem interactions, `random` for generating random sequences).
2. **Function `generate_test_cases`**:
   - Creates the specified directory if it does not exist.
   - Loops to generate the required number of test cases.
   - For each test case:
     - Generates a sequence of length `max_length` containing random 'F' and 'T'.
     - Ensures `n` (number of changes) is within the permissible range (`1 ≤ n ≤ 50`).
     - Writes the generated command string and `n` to the respective file.
3. **Main Execution**:
   - Uses `argparse` to handle command line arguments for directory, number of cases, and maximum length.
   - Executes `generate_test_cases` with the provided arguments.

### How to Use the Script
Save the script in a file, for example `generate_tests.py`, and run it from the command line:

```bash
python generate_tests.py path_to_directory --cases 10 --max_length 100
```

- `path_to_directory`: the directory where the test cases will be saved.
- `--cases 10`: optional argument to specify the number of test cases to generate, default is `10`.
- `--max_length 100`: optional argument to specify the maximum length of the command string, default is `100`.

This setup will create several files (`test_01.in`, `test_02.in`, ...) in the specified directory, each containing a random sequence of `F` and `T` commands and a number of changes `n`. This script primarily aims to stress-test the given solutions and check for efficiency under maximum constraints.
